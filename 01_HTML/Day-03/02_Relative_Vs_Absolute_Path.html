Relative Vs Absolute Path

Embed image using Relative path

The image in this example is embedded by using a relative path from the local device. This means that instead of providing the full directory path or an external URL, the image is referenced in relation to the location of the current HTML file. This approach is commonly used in web development when all the necessary files are located within the same project directory.

Embed image using Absolute path

In this example, the image is embedded by using an absolute path, where the source of the image is specified as a complete URL. This means that instead of referencing a file from the local device or project directory, the image is being fetched directly from a specific location on the web. This method is particularly useful when you need to include resources that are hosted on external websites, ensuring that the image can be accessed consistently across different devices and environments
